# LiteLLM

## Summary

Provider routing and gateway layer for teams that need one interface across many LLM APIs.

## Best for

- multi-provider deployments
- fallback and routing strategies
- centralized logging and cost control

## Why it matters

Not a full agent framework, but a very practical piece of builder infrastructure once you move beyond a single model provider.
