# Getting Started

## For readers

Start in `README.md`, then browse the collection that matches your use case:

- `content/skills/`
- `content/prompts/`
- `content/instructions/`
- `content/mcp/`
- `content/frameworks/`
- `content/evals/`
- `content/workflows/`
- `content/examples/`

If you want a faster overview first, check:

- `content/indexes/featured.md`
- `content/indexes/landscape.md`
- `content/indexes/by-scenario.md`

Each entry should tell you:

- What problem it solves
- When to use it
- What to copy or adapt

## For contributors

1. Pick the correct category.
2. Copy the matching template from `templates/`.
3. Fill in `README.md` and `meta.yml`.
4. Validate metadata locally.
5. Open a pull request.

## Design rule

`content/` stores finished resources. `templates/` stores blank authoring formats.
